Для правильной разработки ячеек статической сетки (блоков) и сущностей следует соблюдать некоторые условия.
`
Для ячеек статической сетки (блоков):
- класс должен быть отнаследован от StaticGridCell
- файл с кодом следует располагать в директории "static_grid_cells\"
- в классе SaveSymbolRegister в методе __init_static_grid_cell_dist необходимо задать соответствие между классом и символом, обозначающим его в файле структуры уровня
- если это препятствие, то класс должен быть отнаследован от Obstacle
- для удаления ячеек во время работы следует использовать метод Level.delete_static_grid_cell
- для добавления новых ячеек во время работы следует использовать метод Level.add_new_static_grid_cell
`
Для сущностей:
- класс должен быть отнаследован от Entity
- файл с кодом следует располагать в директории "entities\"
- в классе SaveSymbolRegister в методе __init_entity_dict необходимо задать соответствие между классом и символом, обозначающим его в файле структуры уровня
- для удаления сущностей во время работы следует использовать метод Level.delete_entity
- для добавления новых сущностей во время работы следует использовать метод Level.add_new_entity


Некоторые особенности:
- в теле класса не нужно писать отрисовку самостоятельно
- для вычисления столкновения с другими ячейками статической сетки (блоками) и сущностями можно использовать событие on_collide или методы collide_with и quick_collide_with
- следует понимать, что в SaveSymbolRegister один символ может соответствовать не более, чем одному классу ячейки статической сетки (блоку) и одной сущности


В классе SaveSymbolRegister есть пример регистрации классов, а классы BrickCell и FakeCloud сами являются примерами
